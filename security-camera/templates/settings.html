General

Displayed in the header on all pages. Maximum 80 characters.

Camera

Capture resolution. Higher resolutions show more detail but use more disk space and take longer to encode in movie mode.
{{ cfg.framerate }} fps
Frames analysed per second. Higher values catch faster movement but use more CPU. 10 is a good balance for a security camera.
{{ cfg.saturation }}
Colour intensity. Set to 0.0 for greyscale — recommended for NoIR cameras which have no IR cut filter and produce a purple/blue cast in colour mode.
{{ cfg.brightness }}
Shifts the overall exposure. 0.0 is unchanged. Increase if captures are too dark; decrease if they are washed out.
{{ cfg.contrast }}
Tonal range multiplier. 1.0 is unchanged. Increase to make dark areas darker and bright areas brighter — useful in low-light scenes.
Auto white balance preset. Has no effect when saturation is 0 (greyscale). Tungsten or Indoor can reduce blue cast under IR lighting.

Motion detection

Still — saves a single JPEG the moment motion is detected. Instant and lightweight.
Movie — records a video clip for the configured duration. Requires more CPU and disk space.
{{ cfg.movie_duration | int }} s
How many seconds to record per clip. The cooldown timer starts after the clip finishes, so back-to-back recordings are spaced at least cooldown + duration seconds apart.
{{ cfg.sensitivity_threshold }}
How much a pixel must change (0–255 scale) before it is counted as motion. Lower = more sensitive. Raise this if the camera triggers on lighting changes, shadows, or flickering.
{{ cfg.min_contour_area }} px²
The smallest moving region (in pixels) that counts as motion. Raise this to ignore insects, dust, or other small objects. At 1280×720: ~1 500 ≈ hand, ~5 000 ≈ torso, ~10 000 ≈ full body.
{{ cfg.cooldown_seconds | int }} s
Minimum time between saves. Prevents bursts of near-identical captures during prolonged movement.

Storage

Where captures are saved on disk. The directory is created automatically if it does not exist.
{{ cfg.retention_days }} days
Files older than this are deleted by a nightly cron job at 02:00.
{{ cfg.min_free_pct }}%
When free space drops below this threshold, the oldest captures are deleted automatically before each new save. Must be lower than the target.
{{ cfg.target_free_pct }}%
Once a cleanup is triggered, old files are deleted until this much space is free. Setting this higher than the minimum reduces how often cleanup runs.

Web server

Port the web UI listens on. Changing this requires the web server to restart — the page will reconnect automatically after ~3 seconds.
0.0.0.0 listens on all network interfaces (accessible from any device on the local network). Use 127.0.0.1 to restrict access to the Pi itself.
{{ cfg.latest_count }} per page
How many captures are loaded at a time when scrolling through the gallery.
